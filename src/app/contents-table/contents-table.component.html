Main Page Introduction

Welcome to my website. When I first started working on it I was frequently asked why I wanted one for myself. One of the reasons obviously is that it is fairly common for software developers to create websites to function as a portfolio, especially if they've done front end work. For me it was also really important to have a place where I could be proud of the work I've done, or a place where I could be motivated to build work for the purpose of being shown on my website. One of the primary motivators for me to follow channels like "The Coding Train" and others is the fact that I have a website which can function as a workspace and a showcase I can share with my friends and easily be proud of it. Another purpose this website serves is as a blog of sorts where I can freely express myself about whatever I want. The simplicity of being able to use it as a journal of sorts where I can add pages to journal into about random events of the day, or hopefully someday it will include some of the thoughts and ideas I've had regarding certain topics. As I started to work on this website more I realized I additionally wanted it to be a place I could go to find examples of my code, code that I was proud of, in order to reuse it in places where necessary.

  • Conway's game of life is a clever program that was developed in 1970 by John Conway. It's a pretty clever program that fits into the cellular automata categorization. I was inspired to work on from The Coding Train on youtube. I love doing cool graphics and I especially enjoyed working on a program utilizing a 2d grid. A notable feature of the game of life is that it can be turing complete.
  • Langton's ant is another cellular automata. What's cool about this one is that given infinite space it devolves into a set pattern which is somewhat interesting. It was developed in 1986 by Chris Langton. It's another clever program I was inspired to work on from The Coding Train on youtube.
  • The traveling salesman problem is an interesting one I came across multiple times before finally deciding to code on it. I was again partly inspired by seeing that The Coding Train did a video on it, but also I love the concept of this because a major solution to this problem involved information gathered from observing ant colonies. One of the things that's always fascinating to me is when we can learn complex systems and solutions from nature and the creatures around us. I learned in school that this ant colony optimization was integral in the development of cellular phones which is pretty fascinating. In this particular page I showcase multiple different solutions to the problem.
  • One of my goals in life is to build a 2d turn based strategy game, and so there are a lot of concepts related to that which I have an interest in because I know they will be helpful when I work on that project. The boids algorithm, designed by Craig Reynolds in 1986, is one of those concepts because it provides an option for AI units to be controlled while still providing for somewhat natural movements and flow. I was again introduced to this concept by The Coding Train.
  • Another project I worked on and am fairly proud of is this linked list showcase. One of the things that's seemed pretty difficult for me to find on the internet is a good graphical representation of the data structures. Because of that I felt it was pretty important that I could showcase my knowledge of such by also providing one such representation.
